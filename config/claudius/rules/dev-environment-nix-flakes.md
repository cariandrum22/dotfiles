## Development Environment Policy - Nix Flakes

### Core Principle

**If Nix is available in your environment, use Nix Flakes for all package-level dependencies and development environment management.**

This policy supersedes traditional package managers for development tools and ensures reproducible environments across all team members.

### Decision Tree

```
1. Is Nix installed?
   ├─ Yes → Use Nix Flakes (this policy)
   └─ No → Can Nix be installed?
       ├─ Yes → Install Nix and use Flakes
       └─ No → Fall back to traditional package managers with clear documentation
```

### Coordination with `just`

- Treat `flake.nix` as the canonical declaration of toolchains and shell environments.
- `just` recipes MAY wrap `nix develop`, `nix build`, or other flake entry points for convenience, but MUST NOT duplicate package installation logic outside the flake.
- Use the `build-tool-just.md` policy for orchestrating imperative tasks that cannot be expressed declaratively; note that Nix Flakes hold primacy for reproducibility.

### Required Files

#### 1. `flake.nix` (Root Configuration)

```nix
{
  description = "Development environment for [PROJECT_NAME]";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
    flake-utils.url = "github:numtide/flake-utils";
    
    # Language-specific inputs
    rust-overlay = {
      url = "github:oxalica/rust-overlay";
      inputs.nixpkgs.follows = "nixpkgs";
    };
  };

  outputs = { self, nixpkgs, flake-utils, rust-overlay, ... }:
    flake-utils.lib.eachDefaultSystem (system:
      let
        overlays = [ (import rust-overlay) ];
        pkgs = import nixpkgs {
          inherit system overlays;
        };
        
        # Define tool versions centrally
        toolVersions = {
          nodejs = pkgs.nodejs_20;
          python = pkgs.python311;
          rust = pkgs.rust-bin.stable.latest.default;
        };
        
      in {
        devShells = {
          default = pkgs.mkShell {
            buildInputs = with pkgs; [
              # Core development tools
              just
              git
              gnumake
              direnv
              
              # Language-specific tools
              toolVersions.nodejs
              toolVersions.python
              toolVersions.rust
              
              # Project-specific tools
              # Add your specific tools here
            ];
            
            # Environment setup
            shellHook = ''
              echo "[INFO] Entering Nix development environment"
              echo "[INFO] Tool versions:"
              echo "  Node.js: $(node --version)"
              echo "  Python: $(python --version)"
              echo "  Rust: $(rustc --version)"
              echo "  Just: $(just --version)"
              
              # Source additional setup if exists
              [ -f .envrc.local ] && source .envrc.local
            '';
          };
          
          # Specialized shells
          ci = pkgs.mkShell {
            buildInputs = with pkgs; [
              # Minimal CI environment
              just
              git
            ];
          };
        };
      });
}
```

#### 2. `.envrc` (direnv Integration)

```bash
#!/usr/bin/env bash
# Automatically load Nix environment

if ! has nix; then
  echo "[ERROR] Nix is not installed. Please install Nix first:"
  echo "   curl -L https://nixos.org/nix/install | sh"
  exit 1
fi

# Use flake if available
if [ -f flake.nix ]; then
  use flake
else
  echo "[WARNING] No flake.nix found. Create one with 'nix flake init'"
fi

# Load local overrides if they exist
[ -f .envrc.local ] && source_env .envrc.local
```

### Language-Specific Patterns

#### JavaScript/TypeScript Projects

```nix
devShells.default = pkgs.mkShell {
  buildInputs = with pkgs; [
    # Core JS tools
    nodejs_20
    nodePackages.pnpm
    nodePackages.yarn
    
    # Development tools
    nodePackages.typescript
    nodePackages.typescript-language-server
    nodePackages.prettier
    nodePackages.eslint
    
    # Build tools
    just
    watchexec
  ];
  
  shellHook = ''
    # Setup npm/pnpm/yarn to use project-local installations
    export NPM_CONFIG_PREFIX="$PWD/.npm"
    export PATH="$NPM_CONFIG_PREFIX/bin:$PATH"
    
    # Install dependencies if needed
    if [ -f package.json ] && [ ! -d node_modules ]; then
      echo "[INFO] Installing npm dependencies..."
      npm ci
    fi
  '';
};
```

#### Python Projects

```nix
devShells.default = pkgs.mkShell {
  buildInputs = with pkgs; [
    # Python and package management
    (python311.withPackages (ps: with ps; [
      pip
      setuptools
      wheel
      virtualenv
    ]))
    
    # Python tools
    poetry
    ruff
    mypy
    black
    
    # Development tools
    just
    watchexec
  ];
  
  shellHook = ''
    # Create venv if it doesn't exist
    if [ ! -d .venv ]; then
      echo "[INFO] Creating Python virtual environment..."
      python -m venv .venv
    fi
    
    # Activate venv
    source .venv/bin/activate
    
    # Install dependencies
    if [ -f requirements.txt ] && [ ! -f .venv/.installed ]; then
      pip install -r requirements.txt
      touch .venv/.installed
    fi
  '';
};
```

### Integration with Other Tools

#### Just Integration

Create a `justfile` that assumes Nix environment:

```just
# Check if running in Nix environment
_require-nix:
    #!/usr/bin/env bash
    if [ -z "$IN_NIX_SHELL" ]; then
        echo "[ERROR] Not in Nix shell. Run 'nix develop' first."
        exit 1
    fi

# All recipes depend on Nix environment
build: _require-nix
    cargo build

test: _require-nix
    cargo test
```

#### CI/CD Integration

**GitHub Actions**:
```yaml
name: CI
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Install Nix
        uses: cachix/install-nix-action@v24
        with:
          extra_nix_config: |
            experimental-features = nix-command flakes
            
      - name: Setup Cachix
        uses: cachix/cachix-action@v14
        with:
          name: your-cache-name
          authToken: '${{ secrets.CACHIX_AUTH_TOKEN }}'
          
      - name: Run tests in Nix shell
        run: nix develop --command just test
```

### Best Practices

1. **Version Pinning**:
   ```nix
   # Pin specific versions when needed
   let
     nodejs = pkgs.nodejs_20;
     rust = pkgs.rust-bin.stable."1.75.0".default;
   in ...
   ```

2. **Overlay Usage**:
   ```nix
   # Custom package overrides
   overlays = [
     (self: super: {
       my-custom-tool = super.callPackage ./nix/my-tool.nix {};
     })
   ];
   ```

3. **Multi-Shell Environments**:
   ```nix
   devShells = {
     default = mkShell { ... };
     backend = mkShell { ... };
     frontend = mkShell { ... };
     minimal = mkShell { ... };
   };
   ```

4. **Secrets Management**:
   ```nix
   shellHook = ''
     # Load secrets from secure location
     if [ -f ~/.config/project/secrets.env ]; then
       set -a
       source ~/.config/project/secrets.env
       set +a
     else
       echo "[WARNING] No secrets file found. Some features may not work."
     fi
   '';
   ```

### Migration Guide

#### From Traditional Setup to Nix

1. **Inventory Current Tools**:
   ```bash
   # List all development dependencies
   which -a node python rust cargo just make
   ```

2. **Create Initial Flake**:
   ```bash
   nix flake init
   nix develop
   ```

3. **Test Everything Works**:
   ```bash
   # In Nix shell
   just test  # or your test command
   ```

4. **Update Documentation**:
   ```markdown
   ## Development Setup
   
   This project uses Nix for dependency management.
   
   1. Install Nix: https://nixos.org/download.html
   2. Enable flakes: 
      ```bash
      echo "experimental-features = nix-command flakes" >> ~/.config/nix/nix.conf
      ```
   3. Enter development shell:
      ```bash
      nix develop
      ```
   ```

### Troubleshooting

1. **Flake Not Found**:
   ```bash
   # Initialize flake
   nix flake init
   nix flake update
   ```

2. **Binary Cache Misses**:
   ```bash
   # Build and push to cache
   nix build
   nix copy --to 's3://your-bucket' .#devShells.x86_64-linux.default
   ```

3. **direnv Not Loading**:
   ```bash
   # Allow direnv
   direnv allow .
   ```

### Compliance Checklist

- [ ] `flake.nix` exists and includes all development dependencies
- [ ] `flake.lock` is committed to version control
- [ ] `.envrc` configured for automatic environment loading
- [ ] README documents Nix requirement and setup
- [ ] CI/CD uses Nix shells for consistency
- [ ] No global tool installation instructions (use Nix instead)
- [ ] All team members use same tool versions via Nix
- [ ] Binary cache configured for faster builds
- [ ] Platform-specific dependencies handled correctly
- [ ] Secrets management doesn't expose sensitive data
